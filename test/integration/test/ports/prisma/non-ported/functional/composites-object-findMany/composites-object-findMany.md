# Non-ported — composites-object-findMany

- `packages/client/tests/functional/composites/object/findMany.ts` › `select` — `findMany({ where, select: { content: { select: { text: true } } } })` returns `[{ content: { text } }]` — prisma-next's mongo ORM `select(...fields)` is top-level only; nested composite sub-selection is not expressible.
- `packages/client/tests/functional/composites/object/findMany.ts` › `orderBy` — `findMany({ where, orderBy: { content: { upvotes: { _count: 'desc' } } } })` — prisma-next's mongo ORM orderBy is `Record<field, 1|-1>`; nested `content.upvotes._count` ordering is not expressible.
- `packages/client/tests/functional/composites/object/findMany.ts` › `filter equals` — `findMany({ where: { content: { equals: <object> } } })` — prisma-next's mongo `where` maps the composite field to a bare value (equality via `$eq` on the embedded document); it has no `{ equals: ... }` operator wrapper. (The bare-value form — upstream's `filter equals shorthand` — is the supported mechanism and is ported and passing.)
- `packages/client/tests/functional/composites/object/findMany.ts` › `filter is` — `findMany({ where: { content: { is: { OR: [...] } } } })` — prisma-next's mongo `where` has no `is` composite-relation operator (and no nested `OR`); not expressible.
- `packages/client/tests/functional/composites/object/findMany.ts` › `filter isNot` — `findMany({ where: { content: { isNot: { text } } } })` — prisma-next's mongo `where` has no `isNot` composite-relation operator; not expressible.
- `packages/client/tests/functional/composites/object/findMany.ts` › `filter isSet` — `findMany({ where: { country: { isSet: true } } })` — prisma-next's mongo `where` maps a scalar field to its bare codec-output value (equality only); it has no `{ isSet: true }` operator.
