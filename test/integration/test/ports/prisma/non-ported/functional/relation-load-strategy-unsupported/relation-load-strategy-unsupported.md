# Non-ported — relation-load-strategy-unsupported

**Recurring gap:** prisma-next exposes no `relationLoadStrategy` query option and no `relationJoins` preview-feature concept. Verified by grepping `packages/` (`grep -rin "relationLoadStrategy\|loadStrategy\|relationJoins\|previewFeature"` → zero matches; the only `strategy` hit in `sql-orm-client/src/collection.ts` is the unrelated MTI polymorphism variant). Both suites here assert that `relationLoadStrategy` is rejected — as a type error AND at runtime with "Unknown argument `relationLoadStrategy`". Since the option does not exist in prisma-next, there is nothing to type-reject and nothing to runtime-throw against.

## unsupported-strategy-for-db.ts

Matrix = `allProviders`, but the single test is guarded `testIf(relationJoinsEnabled && !providerSupportsRelationJoins)` — postgres supports relation joins, so this test never runs for postgres upstream; it exists to prove `'join'`/`'query'` rejection on DBs that don't support relation joins (sqlite/sqlserver/mongodb). Non-portable: prisma-next has no `relationLoadStrategy` option to reject, and the case is not exercised on postgres anyway.

- `packages/client/tests/functional/relation-load-strategy-unsupported/unsupported-strategy-for-db.ts` › `using load strategy that is not supported for provider` — rejects `relationLoadStrategy: 'query'` on a provider that doesn't support relation joins (type + runtime) — prisma-next has no `relationLoadStrategy` query option to reject (and the case does not run for postgres).
