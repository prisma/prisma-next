# Slice: sqlite-create-table-adoption — Dispatch plan

_(In-project slice. Spec: [`./spec.md`](./spec.md). Linear: [TML-2859](https://linear.app/prisma-company/issue/TML-2859). Pattern reference: slice 4 [`planner-create-table-adopts-ddl-ast`](../planner-create-table-adopts-ddl-ast/plan.md).)_

Three dispatches. D1 mirrors slice 4's lowerer-plumbing onto the SQLite stack (pure mechanical fan-out, no behaviour change). D2 migrates the one op the slice is named for, with the byte-parity proof. D3 wires the user-facing authoring API symmetric with PG. Each is independently reviewable; together they ship as one PR.

### Dispatch 1: Thread `Lowerer` through the SQLite planner stack (mechanical fan-out)

- **Outcome:** The SQLite planner stack threads a `Lowerer` end-to-end the same way the Postgres stack does after slice 4. `createSqliteMigrationPlanner(lowerer: Lowerer)` requires the arg; `SqliteMigrationPlanner` holds it as `#lowerer`; `TypeScriptRenderableSqliteMigration`'s constructor accepts an optional `lowerer?: Lowerer`; `renderOps(calls, lowerer?: Lowerer)` forwards it; the abstract `SqliteOpFactoryCallNode.toOp(lowerer?: Lowerer): Op` accepts the optional arg; every concrete `*Call` subclass's `toOp` signature is updated to match. `SqliteControlTargetDescriptor.createPlanner(adapter)` (both call sites — `migrations.createPlanner` at `control-target.ts:39` and the `@deprecated` direct method at `:70`) passes the adapter to `createSqliteMigrationPlanner(adapter)`; the `_adapter` param is renamed to `adapter`. All ~17 test call sites of `createSqliteMigrationPlanner()` (across `@prisma-next/target-sqlite` and `@prisma-next/adapter-sqlite`) are updated to pass a real `SqlControlAdapter<'sqlite'>` (constructed via `createSqliteAdapter()`, which structurally satisfies `Lowerer`). **No behaviour change** — every concrete `toOp` body still ignores the new optional arg.
- **Builds on:** The spec's Chosen design (slice 4's proven plumbing pattern on the PG side, mirror as-is). External: the existing `Lowerer` interface at `@prisma-next/family-sql/control-adapter` and the existing `SqlControlAdapter<'sqlite'>` instance available via `createSqliteAdapter()` in `@prisma-next/adapter-sqlite`.
- **Hands to:** `SqliteCreateTableCall.toOp(lowerer)` receives a real `Lowerer` from the planner; the lowerer is reachable everywhere a `*Call.toOp` is called from. The SQLite planner stack is parity-shaped with the Postgres stack but no op uses the lowerer yet.
- **Focus:** Plumbing-only. Each `*Call` body unchanged. Each test stays green (typecheck + `pnpm test:packages -- @prisma-next/target-sqlite @prisma-next/adapter-sqlite`).

**Dispatch-INVEST check.** _Independent_ — produces a usable hand-off (the threaded lowerer reaches `*Call.toOp`). _Negotiable_ — outcome named; the implementer's grep finds every call site. _Valuable_ — D2 cannot migrate `CreateTableCall.toOp` until the lowerer reaches it; this hand-off is the prerequisite, not "preparation only." _Estimable_ — binary: `git grep "createSqliteMigrationPlanner()"` returns zero matches; typechecks + targeted tests green. _Small_ — uniform transformation across one logical unit (the SQLite planner stack). _Testable_ — typecheck + `pnpm test:packages -- @prisma-next/target-sqlite @prisma-next/adapter-sqlite` green.

### Dispatch 2: Migrate `SqliteCreateTableCall.toOp(lowerer)` to the lower path (byte-parity proof)

- **Outcome:** `SqliteCreateTableCall.toOp(lowerer)` builds a `SqliteCreateTable` DDL node via the contract-free `createTable({ table, columns, constraints?, ifNotExists? })` constructor at `@prisma-next/target-sqlite/contract-free`, lowers it through `lowerer.lower(node, { contract: {} })`, and returns the `Op` whose `execute[0].sql` is the lowered string. The `SqliteTableSpec → { columns: col(...)[], constraints?: [...] }` mapping is inline in the method (per the spec's Open Question 1 working position). The method throws a clear error when called without a lowerer (mirroring `PostgresCreateTableCall.toOp`'s error shape, with text updated for SQLite + the new "Pass the control adapter to createSqliteMigrationPlanner" form). The free `createTable(...)` Op-builder at `operations/tables.ts:45` is no longer called from `CreateTableCall.toOp()` (live-path test: `git grep "createTable(this\\.tableName, this\\.spec)"` returns zero matches in `op-factory-call.ts`). The function and the `renderCreateTableSql` it relies on stay in place — both are still called by `recreateTable(...)` at line 171, which is Phase 2.
- **Builds on:** D1's hand-off (the threaded `Lowerer` reaches `SqliteCreateTableCall.toOp`). The existing substrate from slices 1 and 4: the `SqliteCreateTable` node, the contract-free `createTable(...)` constructor, the SQLite adapter constraint-rendering with its 4 byte-parity tests.
- **Hands to:** SQLite `CREATE TABLE` planner-emitted SQL is byte-identical to pre-slice-5 output, but produced via the typed-AST + adapter-lowering path. The end-to-end `*Call.toOp(lowerer)` path is exercised by a new test that asserts byte-parity on representative tables (simple, composite-PK, FK, unique, autoincrement) — adapter-level rendering tests are insufficient; the proof must drive `toOp(lowerer)` end-to-end. `pnpm fixtures:check` green. Migration golden / snapshot tests green with no regeneration.
- **Focus:** One `*Call` migration on one dialect. Type-resolution + codec-hook expansion stay planner-side feeding `col(name, <resolved-type>)` (do not relocate them). `renderCreateTableSql` is not deleted; only one of its two callers stops calling it.

**Dispatch-INVEST check.** _Independent_ — once D1 lands, this dispatch only touches `CreateTableCall` and its test. _Negotiable_ — outcome names the surface; the `SqliteTableSpec → { columns, constraints }` mapping is the implementer's grep + write work. _Valuable_ — closes the slice's named outcome (the planner's `CREATE TABLE` goes through the AST on SQLite). _Estimable_ — binary: `git grep` returns zero matches; the new byte-parity test asserts exact-string equality; `pnpm fixtures:check` green. _Small_ — one method body + one test file. _Testable_ — the new `toOp(lowerer)` byte-parity test + existing SQLite migration golden / snapshot tests + `pnpm fixtures:check`.

### Dispatch 3: `SqliteMigration.createTable({...})` authoring method + drop free re-export

- **Outcome:** `SqliteMigration` (the abstract base class at `packages/3-targets/3-targets/sqlite/src/core/migrations/sqlite-migration.ts`) gains a constructor that builds and holds a `controlAdapter: SqlControlAdapter<'sqlite'> | undefined` from `stack?.adapter` and a protected `createTable({ table, columns, constraints?, ifNotExists? })` method that instantiates `SqliteCreateTableCall(...)` and calls `.toOp(this.controlAdapter)` — symmetric with `PostgresMigration.createTable` at `postgres-migration.ts:79-94` and `postgres-migration.ts:53-54`. The free `createTable` is removed from `packages/3-targets/3-targets/sqlite/src/exports/migration.ts` (the line `export { createTable, dropTable, recreateTable } from '../core/migrations/operations/tables';` collapses to `export { dropTable, recreateTable } from ...`). The SQLite facade re-export test at `packages/3-extensions/sqlite/test/migration/re-export.test.ts` is updated to remove (or replace with a "method-on-Migration" comment) the assertion for free `createTable` — mirroring slice 4's commit `0d09f8b0b` for the PG facade. User-facing API change is recorded via `record-upgrade-instructions` at PR-write time (the new authoring form is `this.createTable({...})`).
- **Builds on:** D2's hand-off (the migrated `SqliteCreateTableCall.toOp(lowerer)` is the implementation `Migration.createTable` delegates to). The PG pattern reference at `postgres-migration.ts:79-94` for the method shape.
- **Hands to:** User-edited SQLite `migration.ts` files use `this.createTable({...})` for new code; the free `createTable` import path returns "not exported," which the upgrade-instructions PR-time note documents. The slice DoD is met.
- **Focus:** User-facing authoring API symmetric with PG. Touches `sqlite-migration.ts`, `exports/migration.ts`, and the facade `re-export.test.ts`. Does NOT touch the planner-internal code (D1 + D2 handled that).

**Dispatch-INVEST check.** _Independent_ — once D2 lands, this dispatch only adds a method + drops one export + updates one test. _Negotiable_ — outcome names the surfaces; the implementer pattern-matches against `PostgresMigration.createTable`. _Valuable_ — the user-facing API change is the externally-visible payoff of the slice; without it, SQLite is in a "planner adopts AST but user-facing API still uses free string-glued builder" mismatch. _Estimable_ — binary: the new method exists with the correct signature; the free `createTable` export is gone; the facade re-export test passes; downstream consumers' upgrade instruction is recorded. _Small_ — three small file edits. _Testable_ — the facade re-export test, plus typecheck (the abstract method signature is structural), plus the SQLite migration tests (which now exercise `this.createTable` end-to-end).

## Handoff contract — linearity + DoD completeness

- **Linearity.** D2's `builds on` = D1's hand-off (the threaded `Lowerer`). D3's `builds on` = D2's hand-off (the migrated `SqliteCreateTableCall.toOp` is what `Migration.createTable` delegates to). Linear: each dispatch's `builds on` references the immediately-prior dispatch's `hands to`.
- **DoD completeness.** The slice spec lists three DoD items:
  1. _`CreateTableCall.toOp(lowerer)` lowers via the adapter; live-path grep zero._ → satisfied by **D2**.
  2. _Byte-identical SQL via a `toOp(lowerer)` test; goldens + `fixtures:check` green._ → satisfied by **D2**.
  3. _Free `createTable` export removed; facade re-export test updated._ → satisfied by **D3**.
  Every DoD item is reachable from the dispatch sequence.

## Notes for the build loop

- **Test-side regression to watch.** Slice 4's late-surfacing failure mode was that the adapter-postgres test suite passed the *family* to `createPlanner` (old API) and silently broke at runtime with `lower.lower is not a function` — `pnpm build` didn't catch it because per-package builds don't typecheck tests. The same shape exists here: the SQLite planner stack has ~17 test call sites that need updating in D1. The D1 brief should explicitly require running `pnpm --filter @prisma-next/target-sqlite --filter @prisma-next/adapter-sqlite typecheck` AND `... test` before declaring done.
- **`renderCreateTableSql` stays.** D2's grep gate is "no live-path call from `CreateTableCall.toOp`," not "function deleted." The function and `recreateTable`'s call to it stay; Phase 2's `RecreateTable` slice removes the last caller and the function together.
