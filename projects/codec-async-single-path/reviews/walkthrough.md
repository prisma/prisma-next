# Walkthrough — single-path async codec runtime

## Intent

Make the public `Codec` interface uniformly Promise-returning at the query-time boundary so the runtime can always-await — regardless of whether a given codec was authored sync or async — without putting any sync/async discriminator on the public surface. Codec authors and ORM users both work in plain `T`: authors write whichever style fits the codec, users get plain `T` rows out and accept plain `T` writes in. Because no public-surface discriminator was introduced, a synchronous fast path stays open as a non-breaking additive future opt-in (`codecSync()`).

## Change map

Touched packages, grouped by domain:

- **Framework** — `packages/1-framework/1-core/framework-components/`. Public `Codec` base interface widened to 5 generics with `TOutput = TInput` default; query-time methods (`encode` / `decode`) Promise-returning, build-time methods (`encodeJson` / `decodeJson` / `renderOutputType`) stay synchronous.
- **SQL DSL + runtime**:
  - `packages/2-sql/4-lanes/relational-core/` — `codec()` factory accepts sync, async, or mixed author functions and lifts them uniformly to Promise-returning.
  - `packages/2-sql/5-runtime/` — `encodeParams` / `decodeRow` / `decodeField` reshaped onto single-armed always-await + `Promise.all` concurrent dispatch; `RUNTIME.ENCODE_FAILED` / `RUNTIME.DECODE_FAILED` envelopes with structured details and `cause` chaining; build-time `validateContract` / `postgres({...})` regression-locked synchronous.
- **ORM lane** — `packages/3-extensions/sql-orm-client/`. Verification-only: read and write surfaces still expose plain `T`; one shared field type-map across read and write surfaces; `collection-dispatch.ts` documented as codec-agnostic.
- **Mongo family** — `packages/2-mongo-family/` and `packages/3-mongo-target/`:
  - `mongoCodec()` factory aligned with SQL `codec()` — sync-or-async authoring, uniform Promise-returning lift, `MongoCodec` aliases `BaseCodec` exactly (5 generics matching, `TOutput = TInput` default).
  - `MongoAdapter.lower(plan): Promise<AnyMongoWireCommand>`; `resolveValue` async + `Promise.all` over object children and array elements; multi-task lowering inside `lower()` (filter+update tuples, `insertMany` documents, aggregate pipeline stages) also concurrent.
  - Build-time `validateMongoContract` and `createMongoAdapter()` regression-locked synchronous.
- **Adapters / extensions** — `packages/2-sql/6-adapters/postgres/` and `packages/2-sql/6-adapters/sqlite/` codec test casts narrowed to Promise-returning; `packages/3-extensions/pgvector/` test fixtures aligned with the new codec interface.
- **Docs & ADRs**:
  - `docs/architecture docs/adrs/ADR 204 - Single-Path Async Codec Runtime.md` — new, canonical decision record.
  - `docs/architecture docs/adrs/ADR 030 - Result decoding & codecs registry.md` — partial-supersession pointer for the query-time method signatures.
  - Five package READMEs (`framework-components`, `relational-core`, `sql-orm-client`, `mongo-codec`, `mongo-adapter`) refreshed to narrate the async-at-the-boundary contributor-facing story. The `mongo-lowering` README also updated to reflect the new `lower()` signature.

## The story

1. **Public contract first.** Reshape the framework-agnostic `Codec` interface to 5 generics with `TOutput = TInput` default; make `encode` / `decode` Promise-returning at the boundary; keep `encodeJson` / `decodeJson` / `renderOutputType` synchronous. Land the `codec()` factory accepting sync-or-async author functions and uniformly lifting them via `async (x) => fn(x)`. Tests come first per `AGENTS.md` § Golden Rules; the source change is then small and follow-on. The 4-vs-5 generic question (split now vs. defer) is resolved in favor of splitting now so the asymmetric `TInput ≠ TOutput` case is expressible from the start; existing 4-generic call sites continue to work via the default.

2. **SQL runtime always-awaits.** `encodeParams` and `decodeRow` become async; each builds a `tasks: Promise<unknown>[]` array and awaits a single `Promise.all` per site (encode-once-per-statement, decode-per-row). `decodeField` is the single-armed per-cell dispatch path: one `await codec.decode(wireValue)`, optional sync `validateJsonValue` against the resolved value, return plain decoded value. Failures wrap into `RUNTIME.ENCODE_FAILED` / `RUNTIME.DECODE_FAILED` envelopes with `{label, codec, paramIndex}` / `{table, column, codec}` (or alias-fallback) details and `cause` chaining; the pre-existing `RUNTIME.JSON_SCHEMA_VALIDATION_FAILED` envelope is preserved verbatim — `wrapDecodeFailure` re-raises the original envelope when the thrown error's code matches. No marker, no `TRuntime`, no predicates, no plan-walker, no `instanceof Promise` guards. Build-time `validateContract` / `postgres({...})` are pinned synchronous at type and runtime levels.

3. **ORM lane carries plain `T` through unchanged.** The runtime always-await + the factory's sync→async lift already make the ORM lane structurally correct; this milestone pins that at the type level and verifies it under live Postgres. 21 type tests assert `DefaultModelRow` / `InferRootRow` / `Collection.first()` / `for await ... of c.all()` / `Collection.all().firstOrThrow()` yield plain `T`, and that `CreateInput` / `MutationUpdateInput` / `UniqueConstraintCriterion` / `ShorthandWhereFilter` accept plain `T`. One field type-map shared between read and write surfaces (no read/write split). Live-Postgres integration tests round-trip `pg/vector@1` and `pg/jsonb@1` codecs through `.first()`, `for await`, `create()`, and `update()` and inspect persisted bytes (`select embedding::text` returns `'[0.1,0.2,0.3]'`, `select address` returns the codec's JSON shape). `collection-dispatch.ts` carries a header doc block recording that per-row decoding is owned upstream in `sql-runtime`'s row-yielding async generator and this file never calls codec query-time methods directly.

4. **Mongo cross-family parity.** Propagate the same single-path async-codec design to the Mongo family's encode-side runtime. `MongoAdapter.lower(plan)` becomes `async` and returns `Promise<AnyMongoWireCommand>`; `resolveValue` walks `MongoParamRef` leaves with `await codec.encode(...)` and dispatches array elements + object children concurrently via `Promise.all`. Multi-task lowering inside `lower()` (filter+update tuples for the four update-family ops; `insertMany` documents; aggregate pipeline stages) also dispatches concurrently. Raw command variants bypass codec lowering entirely (raw escape-hatch preserved). Build-time `validateMongoContract` and `createMongoAdapter()` stay synchronous (regression-locked). `MongoCodec` is widened from 4 to 5 generics matching `BaseCodec` exactly so a single `codec({...})` value is structurally usable in both family registries — including the asymmetric `TInput ≠ TOutput` case. Mongo decode-side is intentionally out of project scope: Mongo doesn't decode rows today, and adding one would invent a new subsystem orthogonal to async codecs. ADR 204 records the future-design pattern (mirror SQL's `decodeRow`).

5. **Security tests, ADR canon, README sweep.** Translate PR #375's security tests verbatim — envelope shape with `cause` chaining, JSON-Schema validation runs against the awaited resolved value, seeded-secret-codec end-to-end (real AES-GCM async crypto round-trip through the runtime). PR #375's marker artifacts (`runtime: 'async'`, `kind`, `TRuntime`) are stripped from the translations because this project explicitly forbids them. Three deferrals are recorded as `it.skip` with explicit rationale: PR #375's redaction-trigger spelling test (spec § Non-goals — redaction-spelling is tracked separately); three include-aggregate child-row decoding tests (concrete blocker — the dispatcher does not invoke codecs on `jsonb_agg` child cells today). PR #375's promise-valued child-cell test is dropped, not skipped — it asserts the inverse of the design's plain-`T` guarantee. Author ADR 204 — Single-Path Async Codec Runtime in canonical location with the four PR #375 review critiques folded into Context (cost-in-the-wrong-place, no-walk-back, wrong-seam, cross-family-portability) and the seven NFR #5 walk-back constraints transcribed verbatim. Add a partial-supersession pointer to ADR 030 scoped to the query-time method signatures only. Refresh five package READMEs to narrate the async-at-the-boundary story.

## Compatibility / migration / risk

- **Walk-back stays open.** The seven NFR #5 walk-back constraints are verified absent across the project: no per-codec async marker; no `codecSync` / `codecAsync` variants; no `isSync*` predicates; no conditional return types on Promise-returning methods; no `TRuntime` generic on `Codec` or `MongoCodec`; no async-dependent author-surface docs; no async-dependent build-time guarantees. ADR 204 transcribes the seven constraints verbatim. The future `codecSync()` opt-in remains a non-breaking additive change.
- **Backward-compatible widening.** Every existing call site of `codec()` and `mongoCodec()` uses the symmetric `TInput=TOutput=TJs` form; the `TOutput = TInput` default collapses the new 5-generic shape to the old 4-generic shape for those call sites. No source change required in any consumer (built-in codecs, adapters, runtime, integration fixtures).
- **Build-time stays sync.** `validateContract` / `validateMongoContract` / `postgres({...})` / `createMongoAdapter()` are regression-locked at the type and runtime levels. If any drifts to Promise-returning, both type tests and `expect(typeof thenable.then).toBe('undefined')` checks fail loudly.
- **`Promise.all` failure semantics are partial fail-fast.** The first rejected codec dispatch surfaces as the error envelope; remaining dispatched bodies still run to completion (cannot be cancelled in standard `Promise.all`) but their outputs are discarded. From the user's point of view this matches the prior synchronous `for`-loop's "throw on first failure" behavior.
- **Allocation pressure now realized.** Every codec dispatch allocates one Promise per cell; for a query returning N rows × M cells, that's N × M Promise allocations. The walk-back path (`codecSync()` + predicates as a future additive opt-in) is preserved precisely because of this cost — the design accepts the allocation cost today in exchange for keeping the public surface single-path and async-friendly.
- **Inline `Codec`-shaped test fixtures required mechanical updates** across `framework-components`, `sql-contract-ts`, `cli`, `integration-tests`, `pgvector`, `adapter-postgres`, `adapter-sqlite`. Recipe: `decode: (wire) => wire` → `decode: async (wire) => wire`, add `encode: async (value) => value` when missing, prefix call sites with `await`, convert sync `expect(...).toThrow(...)` to `await expect(...).rejects.toThrow(...)`. No production-source escape hatches needed; the m1 factory's lift normalizes any combination of sync and async author functions.
- **No public re-exports / shims for backwards compatibility.** Per `AGENTS.md` § Golden Rules. The legacy `MongoCodecJsType<T>` extractor was replaced by `MongoCodecInput<T>` / `MongoCodecOutput<T>` (mirroring SQL's `CodecInput<T>` / `CodecOutput<T>`) without a backcompat alias.
- **No spec changes required.** Every acceptance criterion is met or scope-noted within the existing spec.
- **Pre-existing flakes carried over (informational, not introduced).** `MongoMigrationRunner` CAS flake in `target-mongo` and a 100ms timeout on `test/authoring/side-by-side-contracts.test.ts` in the integration suite. Both are out of project scope.

2. **Future projects identified during this work.**

   - **Mongo row decoding.** The next-family extension; ADR 204 directs implementers to mirror SQL's `decodeRow` pattern (single-armed always-await + `Promise.all` per-cell + await-before-yield).

## Non-goals / intentionally out of scope

- **Synchronous fast path.** A future `codecSync()` opt-in is preserved as an additive non-breaking change; not landed in this project.
- **Mongo decode-side codec dispatch.** Mongo doesn't decode rows today; out of project scope by design. ADR 204 § Cross-family scope notes records the future pattern.
- **Include-aggregate child-codec dispatch.** Orthogonal ORM feature; deferred with `it.skip` placeholders.
- **Redaction-trigger spelling.** Orthogonal redaction-policy work; deferred with `it.skip`.
- **No backwards-compatibility shims.** Per `AGENTS.md` § Golden Rules.
