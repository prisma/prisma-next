### F-1 тАФ тЪая╕П High тАФ `@default(uuid())` silently clobbers an explicit bare `Uuid` field type to `char(36)`; named-type spelling survives

**Scenario:** 5 тАФ Exploratory charter ("do the classic attribute defaults still compose with the new types?" probe)
**Step:** exploratory probe at ~minute 3, isolated at ~minute 13
**Oracle:** the slice's parity principle тАФ a bare native scalar in field position emits the same `{ codecId, nativeType, typeParams }` as every other spelling of the same declaration; scenario 1 proved field position and named-type position agree for all fourteen spellings.
**Observed:**

```
$ cat qa-scratch-5/schema.prisma
// use prisma-next

types {
  TUuid = Uuid
}

model E {
  id TUuid @id @default(uuid())
}

}

```

**Expected (per script/spec):** `F.id` should carry `pg/uuid@1` / `uuid` like `E.id` (modulo `typeRef`) тАФ the user explicitly declared `Uuid`. No diagnostic was emitted; the override is silent.
**Reproduction:**

- `git rev-parse HEAD` тЖТ `d2b5cab9213d7ee2ff5384ce6b07e4c939294c60`
- `git status` at failure тЖТ clean (baseline untracked only)
- Mutated files: none in tree; repro schema + emitted contract preserved at `manual-qa-reports/artefacts/F-1/` (`schema.prisma`, `contract-uuid-default-clobber.json`)
- Exact command: `./node_modules/.bin/prisma-next contract emit --config qa-scratch-5/config.ts --output-path qa-scratch-5/out`
  **Notes:** Mechanism located at `packages/2-sql/2-authoring/contract-psl/src/psl-field-resolution.ts:510-519`: when a lowered on-create generator exists and the field does **not** use a named type, `generatorDescriptor.resolveGeneratedColumnDescriptor` unconditionally replaces the field's descriptor. The generator-override path predates this PR (on `main`, `String @id @default(uuid())` тЖТ `char(36)` is unremarkable because `String` declares no native type). What's new: this slice makes `Uuid` a first-class *explicit* field type, and the override now silently contradicts an explicit declaration тАФ the same "silent wrong native type" trap scenario 3 exists to guard against, and inconsistent between the two authoring positions the slice promises are equivalent. Suggested scope: skip the generator override when the field's type constructor resolved through `NATIVE_TYPE_SPECS` (mirroring the existing named-type exemption). If the orchestrator judges generator-vs-declared-type precedence a design question (e.g. whether the uuid() runtime generator may store into a native `uuid` column), downgrade disposition to ЁЯОл ticket тАФ but the silent contradiction should not ship unacknowledged.
